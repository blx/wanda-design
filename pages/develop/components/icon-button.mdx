Buttons can be used to communicate and immediately trigger actions a user can take in an application, like submitting a form, canceling a process, or creating a new object. Buttons can also be used to take a user to a new location, like another page inside of a web application, or an external site such as help or documentation.

## Usage

The very basic usge of the component doesn't require any property. If you want to use a `regular` and `primary` button you just have to drop the component inside your code. The visual appearance of the button can be changed by using the `type` property.

<LiveArea direction="row">
  <IconButton icon="plus" />
  <IconButton icon="bell" type="secondary" />
  <IconButton icon="message" type="flat" />
  <IconButton icon="play" disabled />
</LiveArea>

```jsx
<IconButton icon="plus" />
<IconButton icon="bell" type="secondary" />
<IconButton icon="message" type="flat" />
<IconButton icon="play" disabled />
```

## Sizes

Sometimes you don't have enought space to add a regular button, and in the same way sometimes you need a bigger button to keep the visual hierarchy. For these cases you can use the `size` property to change the size of the button, by choosing from three options: `big`, `regular` (default), `small`:

<LiveArea direction="row">
  <IconButton size="big" icon="plus" />
  <IconButton icon="bell" type="secondary" />
  <IconButton size="small" icon="message" type="flat" />
</LiveArea>

```jsx
<IconButton size="big" icon="plus" />
<IconButton icon="bell" type="secondary" />
<IconButton size="small" icon="message" type="flat" />
```

## Dynamic tag

If you need a link which must look like a button you can change the element used by the `<IconButton>` from `<button>` to `<a>` by using the `as` property. In the following example the button will render a link element as wrapper.

<LiveArea direction="row">
  <IconButton icon="bell" as="a" href="https://design.wonderflow.ai" />
</LiveArea>

```jsx
<IconButton as="a" href="https://design.wonderflow.ai" />
```
